# Buttons

Buttons allow users to trigger actions with a single tap or click. Our button system provides a consistent interactive element across both desktop and mobile interfaces.

<Note>
The desktop and mobile buttons are the same but use different typography. Desktop uses the "Helvetica Neue LT Pro" while Mobile uses "Gilroy".
</Note>

**[View in Design System →](https://www.figma.com/design/r4pyHXCJ6L0lwcRhkHDXXc/Giv.-Design-System?node-id=1622-625)**

## Button Props

| Prop Name | Type | Default | Description |
|-----------|------|---------|-------------|
| `variant` | string | `'primary'` | Button style variant (`'primary'`, `'secondary'`, `'tertiary'`, `'ghost'`, `'danger'`, `'success'`) |
| `size` | string | `'md'` | Button size (`'sm'`, `'md'`, `'lg'`) |
| `disabled` | boolean | `false` | Whether the button is disabled |
| `isLoading` | boolean | `false` | Whether to show loading state |
| `type` | string | `'button'` | HTML button type (`'button'`, `'submit'`, `'reset'`) |
| `ariaLabel` | string | - | Aria label for accessibility (required if button has no text) |
| `iconPosition` | string | `'left'` | Position of icon if present (`'left'`, `'right'`) |
| `hasIcon` | boolean | `false` | Whether the button has an icon |
| `fullWidth` | boolean | `false` | Whether the button should take up full width of its container |

## Button Variants

Our design system includes multiple button variants for different contexts and emphasis levels.

### Primary Button

Use for primary actions that we want users to focus on.

```typescript
<button gvButton variant="primary">Primary Action</button>
```

### Secondary Button

Use for secondary actions that are important but not the primary focus.

```typescript
<button gvButton variant="secondary">Secondary Action</button>
```

### Tertiary Button

Use for less important actions or in areas with space constraints.

```typescript
<button gvButton variant="tertiary">Tertiary Action</button>
```

### Ghost Button

Use for the least important actions or in busy interfaces where visual weight needs to be minimized.

```typescript
<button gvButton variant="ghost">Ghost Action</button>
```

### Danger Button

Use for destructive actions that might have significant consequences.

```typescript
<button gvButton variant="danger">Delete Item</button>
```

### Success Button

Use for confirming positive actions.

```typescript
<button gvButton variant="success">Complete Order</button>
```

## Usage Guidelines

### Do's

- Use button text that clearly describes the action
- Use primary buttons for the main action in a section
- Maintain hierarchy with proper use of variants
- Keep button text concise (1-3 words is ideal)
- Use sentence case for button text
- Use loading states for actions that take time

<Frame>
  <img src="/images/button-dos.png" alt="Examples of correct button usage" />
</Frame>

### Don'ts

- Avoid using too many primary buttons on one screen
- Don't use vague text like "Click Here" or "Submit"
- Don't use all caps or title case for button text
- Don't disable buttons without providing a reason
- Avoid using buttons when links would be more appropriate

## Accessibility

Our button component is designed to be fully accessible:

- **Color contrast**: All buttons meet WCAG AA standards (4.5:1 for normal text, 3:1 for large text)
- **Focus states**: Visible focus indicators for keyboard navigation
- **Disabled states**: Maintain sufficient contrast while clearly indicating unavailability
- **Loading states**: Communicate status to screen readers via aria-busy attribute
- **Touch targets**: All buttons meet minimum size requirements (at least 44×44px)
- **Screen readers**: Proper ARIA roles and labels are implemented

```typescript
// Accessible button with loading state
<button 
  gvButton
  variant="primary"
  [disabled]="isLoading"
  [attr.aria-busy]="isLoading"
  aria-label="Submit form">
  <span *ngIf="isLoading" class="sr-only">Loading, please wait</span>
  <span *ngIf="!isLoading">Submit</span>
</button>
```

```css
  /* Additional variables for other variants */

```

## Migration Guide

If you're migrating from our previous button system, here are the key differences:

| Previous Feature | New Approach | Notes |
|-----------------|--------------|-------|
| `.btn-default` class | `variant="secondary"` prop | Secondary is now the default alternative to primary |
| Size modifiers | `size` prop | Standardized to 'sm', 'md', 'lg' |
| Icon buttons | `hasIcon` and `iconPosition` props | Improved accessibility and layout control |
| Inline styles | CSS variables | For theming and customization |

### Breaking Changes

- The `size="xs"` option has been removed
- Button padding has been standardized and may affect layouts
- The `rounded` prop has been removed; all buttons now have consistent rounded corners
