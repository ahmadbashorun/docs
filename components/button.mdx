# Button Component

The Button component is a versatile UI element that supports multiple states, variants, and sizes to accommodate different design needs and user interactions.

## Basic Usage

```html
<button 
  class="btn btn-primary" 
  [disabled]="isDisabled"
  (click)="handleClick()">
  Button Text
</button>
```

## Variants

### Primary

### Secondary

```html

```

### Small


## Implementation Guide

### Angular Component Properties

| Property    | Type      | Default   | Description |
|-------------|-----------|-----------|-------------|
| variant     | string    | 'primary' | Button style variant ('primary', 'secondary', 'tertiary', 'destructive') |
| size        | string    | 'md'      | Button size ('sm', 'md', 'lg') |
| disabled    | boolean   | false     | Whether the button is disabled |
| isLoading   | boolean   | false     | Whether to show loading state |
| type        | string    | 'button'  | HTML button type ('button', 'submit', 'reset') |
| ariaLabel   | string    | undefined | Aria label for accessibility |

### Events

| Event       | Description |
|-------------|-------------|
| clicked     | Emitted when button is clicked |
| focusChange | Emitted when button focus state changes |

### CSS Variables

```css
:root {
  --btn-primary-bg: #4F46E5;
  --btn-primary-text: #FFFFFF;
  --btn-primary-border: transparent;
  --btn-primary-hover-bg: #4338CA;
  --btn-primary-active-bg: #3730A3;
  --btn-primary-disabled-bg: #EEF2FF;
  --btn-primary-disabled-text: #A5B4FC;
  
  /* Additional variables for other variants */
}
```

## Accessibility Considerations

- Ensure proper contrast ratios between text and background colors
- Use appropriate aria attributes for non-standard buttons
- Keyboard navigation support with visible focus states
- Loading states should announce status change to screen readers
